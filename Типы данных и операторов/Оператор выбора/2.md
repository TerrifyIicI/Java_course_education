## Практика 2 — Отсутствие break (fall-through)

### Код

```java
case 1:
    i = 1;
    break;
case 2:
    i = 2;
case 3:
    i = 3;
    break;
```

### Что произойдёт

Если `break` отсутствует, выполнение **проваливается** (fall-through) в следующий `case`.

| Значение value | Результат i | Объяснение |
|----------------|-------------|------------|
| 1 | 1 | Выполняется `case 1`, затем `break` |
| 2 | 3 | Выполняется `case 2` (i=2), затем проваливается в `case 3` (i=3) |
| 3 | 3 | Выполняется `case 3`, затем `break` |

### Когда fall-through полезен

```java
switch (month) {
    case 1:
    case 3:
    case 5:
    case 7:
    case 8:
    case 10:
    case 12:
        days = 31;
        break;
    case 4:
    case 6:
    case 9:
    case 11:
        days = 30;
        break;
    case 2:
        days = 28;
        break;
}
```

Здесь несколько `case` выполняют одно и то же действие, поэтому `break` ставится только в конце группы.
