## Практика #3 — Зачем нужны Reader и Writer

Иерархии `Reader`/`Writer` нужны для работы с **символьными** (текстовыми) потоками, в отличие от **байтовых** `InputStream`/`OutputStream`.

### Основные отличия

**InputStream/OutputStream:**
- Единица данных — байт (8 бит)
- Назначение — бинарные данные (изображения, архивы, файлы)
- Не учитывает кодировку

**Reader/Writer:**
- Единица данных — символ (16 бит, Unicode)
- Назначение — текстовые данные
- Поддерживает различные кодировки (UTF-8, UTF-16, Windows-1251 и др.)

### Зачем это нужно

1. **Поддержка Unicode** — символы могут занимать более одного байта (UTF-8, UTF-16)

2. **Корректная работа с текстом** — автоматическое преобразование байтов в символы с учётом кодировки

3. **Удобство** — методы работают с `char` и `String`, а не с `byte`

```java
// Байтовый поток — читает байты
InputStream is = new FileInputStream("file.txt");
int b = is.read();  // один байт

// Символьный поток — читает символы с учётом кодировки
Reader reader = new FileReader("file.txt");
int ch = reader.read();  // один символ (может быть несколько байт)
```
