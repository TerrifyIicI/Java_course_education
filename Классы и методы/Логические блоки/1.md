## Практика 1 — Порядок выполнения логических блоков

### Код

```java
public class A {
    {
        System.out.println("logic (1) id= " + this.id);
    }

    static {
        System.out.println("static logic");
    }

    private int id = 1;

    public A(int id) {
        this.id = id;
        System.out.println("ctor id= " + id);
    }

    {
        System.out.println("logic (2) id= " + id);
    }
}

public class Main {
    public static void main(String[] args) {
        new A(100);
    }
}
```

### Вывод

```
static logic
logic (1) id= 0
logic (2) id= 1
ctor id= 100
```

### Объяснение

**1. `static logic`**
Статический блок выполняется первым при загрузке класса, ещё до создания объекта.

**2. `logic (1) id= 0`**
Первый логический блок выполняется до инициализации поля `id = 1`. В этот момент поле `id` имеет значение по умолчанию для `int`, то есть `0`.

**3. `logic (2) id= 1`**
Второй логический блок выполняется после строки `private int id = 1`. Поле `id` уже инициализировано значением `1`.

**4. `ctor id= 100`**
Конструктор вызывается последним и присваивает полю `id` значение `100`.

---

## Порядок инициализации объекта

1. Статические блоки (один раз при загрузке класса)
2. Логические блоки и инициализация полей (в порядке объявления)
3. Конструктор
