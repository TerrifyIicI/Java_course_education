# Аннотации в Java
## Практика #1 — Стандартные аннотации

### @Override

**Назначение:** Указывает, что метод переопределяет метод суперкласса или реализует метод интерфейса.

**Зачем нужна:**
- Помогает обнаружить ошибки на этапе компиляции (опечатки в имени метода, неправильные параметры)
- Делает код более читаемым — явно видно, что метод переопределён

```java
/**
 * Пример использования @Override
 *
 * @author Arina
 */
class Animal {
    public void makeSound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    @Override
    public void makeSound() {  // Правильное переопределение
        System.out.println("Woof!");
    }
    
    // @Override
    // public void makeS0und() {  // Ошибка компиляции — метода с таким именем нет в суперклассе
    //     System.out.println("Woof!");
    // }
}
```
### @Deprecated

**Назначение:** Помечает элемент как устаревший. Предупреждает разработчиков, что элемент не рекомендуется использовать и может быть удалён в будущих версиях.

**Зачем нужна:**
- Информирует о необходимости использовать альтернативные методы
- Компилятор выдаёт предупреждение при использовании устаревшего элемента

```java
/**
 * Пример использования @Deprecated
 *
 * @author Arina
 */
class Calculator {
    
    /**
     * @deprecated Используйте {@link #addNumbers(int, int)} вместо этого метода
     */
    @Deprecated
    public int add(int a, int b) {
        return a + b;
    }
    
    // Новый рекомендуемый метод
    public int addNumbers(int a, int b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        // Предупреждение компилятора: 'add(int, int)' is deprecated
        int result1 = calc.add(2, 3);
        
        // Рекомендуемый способ
        int result2 = calc.addNumbers(2, 3);
    }
}
```
### @SuppressWarnings

**Назначение:** Подавляет предупреждения компилятора указанного типа.

**Зачем нужна:**
- Убирает предупреждения, когда разработчик осознанно использует определённый код
- Делает вывод компилятора чище, убирая ожидаемые предупреждения

**Основные типы предупреждений:**
- `"unchecked"` — непроверенные операции с generics
- `"deprecation"` — использование устаревших элементов
- `"unused"` — неиспользуемые переменные
- `"all"` — все предупреждения

```java
import java.util.ArrayList;
import java.util.List;

/**
 * Пример использования @SuppressWarnings
 *
 * @author Arina
 */
public class Main {
    
    // Подавление предупреждения о непроверенных операциях с generics
    @SuppressWarnings("unchecked")
    public void uncheckedExample() {
        List list = new ArrayList();  // raw type
        list.add("Hello");            // unchecked call
    }
    
    // Подавление предупреждения об использовании устаревшего метода
    @SuppressWarnings("deprecation")
    public void deprecationExample() {
        Calculator calc = new Calculator();
        calc.add(2, 3);  // Используем устаревший метод без предупреждения
    }
    
    // Подавление нескольких предупреждений
    @SuppressWarnings({"unchecked", "deprecation"})
    public void multipleWarnings() {
        List list = new ArrayList();
        Calculator calc = new Calculator();
        calc.add(1, 2);
    }
    
    // Подавление предупреждения о неиспользуемой переменной
    @SuppressWarnings("unused")
    public void unusedExample() {
        int unusedVariable = 10;  // Не используется, но предупреждения нет
    }
}
```

- `@Override` - Проверка переопределения метода на этапе компиляции 
- `@Deprecated` - Пометка устаревших элементов
- `@SuppressWarnings` - Подавление предупреждений компилятора
