## Практика 2 — Ключевое слово final в catch

```java
try {
    // ...
} catch (final Exception e) {
    // ...
}
```

### Что означает final?

Ключевое слово `final` означает, что переменную `e` **нельзя переприсвоить** внутри блока catch.

```java
/**
 *
 * @author Arina
 */
public class FinalCatchExample {
    public static void main(String[] args) {
        // Без final — можно переприсвоить
        try {
            throw new Exception("Первое исключение");
        } catch (Exception e) {
            e = new Exception("Другое исключение");  // OK
            System.out.println(e.getMessage());
        }

        // С final — нельзя переприсвоить
        try {
            throw new Exception("Первое исключение");
        } catch (final Exception e) {
            // e = new Exception("Другое исключение");  // Ошибка компиляции!
            System.out.println(e.getMessage());
        }
    }
}
```

### Зачем это нужно:

- **Безопасность** — гарантирует, что исходное исключение не будет случайно заменено
- **Читаемость** — явно показывает намерение не изменять переменную
- **При использовании `|`** — переменная неявно является `final`

```java
// При multi-catch переменная автоматически final
catch (IOException | SQLException e) {
    // e = new Exception();  // Ошибка! Нельзя переприсвоить
}
```
