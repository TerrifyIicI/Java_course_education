## Практика 1 — Назначение основных классов исключений

### Throwable

Базовый класс для всех исключений и ошибок. Содержит информацию о причине исключения, стек вызовов и сообщение об ошибке.

### Error

Критические ошибки, которые **не следует обрабатывать** в программе. Обычно связаны с проблемами JVM или системы.

**Примеры:**
- `OutOfMemoryError` — нехватка памяти
- `StackOverflowError` — переполнение стека
- `VirtualMachineError` — ошибка виртуальной машины

### Exception

Исключения, которые **можно и нужно обрабатывать**. Делятся на проверяемые (checked) и непроверяемые (unchecked).

**Проверяемые исключения** — компилятор требует их обработки:
- `IOException` — ошибки ввода/вывода
- `SQLException` — ошибки работы с базой данных
- `FileNotFoundException` — файл не найден

### RuntimeException

Непроверяемые исключения, которые возникают во время выполнения программы. Компилятор **не требует** их обязательной обработки.

**Примеры:**
- `NullPointerException` — обращение к null
- `ArrayIndexOutOfBoundsException` — выход за границы массива
- `ArithmeticException` — арифметическая ошибка (деление на ноль)
