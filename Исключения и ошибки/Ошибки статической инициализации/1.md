## Практика #1 — Что произойдёт при выполнении

```java
public class App {
    final static int START_COUNTER;
    static {
        START_COUNTER = Integer.parseInt("Y-");
    }
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```

Программа **аварийно завершится** с ошибкой `ExceptionInInitializerError`. Строка `"Hello"` **не будет напечатана**.

### Вывод

```
Exception in thread "main" java.lang.ExceptionInInitializerError
    at App.<clinit>(App.java:5)
Caused by: java.lang.NumberFormatException: For input string: "Y-"
    at java.base/java.lang.NumberFormatException.forInputString(NumberFormatException.java:67)
    at java.base/java.lang.Integer.parseInt(Integer.java:668)
    at java.base/java.lang.Integer.parseInt(Integer.java:786)
    at App.<clinit>(App.java:5)
```

### Пояснение

1. **Статический блок выполняется при загрузке класса** — до вызова метода `main()`

2. **`Integer.parseInt("Y-")`** пытается преобразовать строку `"Y-"` в число, но это невозможно — возникает `NumberFormatException`

3. **Исключение в статическом блоке** оборачивается в `ExceptionInInitializerError` — это критическая ошибка инициализации класса

4. **Класс не может быть загружен** — метод `main()` не выполняется, программа завершается

5. **`ExceptionInInitializerError`** — это подкласс `Error`, а не `Exception`, поэтому его обычно не перехватывают

### Как исправить

```java
/**
 * Исправленный вариант с обработкой исключения
 *
 * @author Arina
 */
public class App {
    final static int START_COUNTER;
    static {
        int temp;
        try {
            temp = Integer.parseInt("Y-");
        } catch (NumberFormatException e) {
            temp = 0;  // Значение по умолчанию
            System.out.println("Ошибка парсинга, используется значение по умолчанию");
        }
        START_COUNTER = temp;
    }
    public static void main(String[] args) {
        System.out.println("Hello");
        System.out.println("START_COUNTER = " + START_COUNTER);
    }
}
```
