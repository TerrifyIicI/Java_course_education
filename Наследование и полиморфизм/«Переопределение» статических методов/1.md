## Практика 1 — Почему динамический полиморфизм не применим к статическим методам

### Код с ошибкой

```java
class A {
    public static void method() {
        System.out.println("Method in A");
    }
}

class B extends A {
    @Override  // Ошибка компиляции!
    public static void method() {
        System.out.println("Method in B");
    }
}
```
### Ответ

Аннотация `@Override` вызовет ошибку компиляции, потому что **статические методы нельзя переопределять**.

### Объяснение

**Динамический полиморфизм** работает так:
- Решение о том, какой метод вызвать, принимается **во время выполнения**
- Выбор зависит от **реального типа объекта**
- Требуется существование объекта

**Статические методы:**
- Принадлежат **классу**, а не объекту
- Вызываются через **имя класса**, а не через ссылку на объект
- Связывание происходит **на этапе компиляции** (статическое связывание)
- Не требуют создания экземпляра класса
