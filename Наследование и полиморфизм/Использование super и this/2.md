## Практика 2 — super при отсутствии членов в промежуточном классе

```java
class A {
    int a;
    void method() {
        System.out.println("Method in A");
    }
}

class B extends A {
    // Поле a и метод method() НЕ определены
}

class C extends B {
    void method() {
        int a = super.a;      // Получит значение из класса A
        super.method();       // Вызовет метод из класса A
    }
}
```
### Объяснение

Если переменная `a` и метод `method()` не определены в классе `B`, то:

- `super.a` — обратится к полю `a` из класса `A` (через цепочку наследования)
- `super.method()` — вызовет метод `method()` из класса `A`

Ключевое слово `super` ищет член в ближайшем суперклассе. Если его там нет, поиск продолжается вверх по иерархии наследования до тех пор, пока член не будет найден.
